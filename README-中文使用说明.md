# 我的研究网站 · 使用说明(大白话版)

这份说明教你三件事:**怎么发报告 / 记交易日志**、**怎么更新价格**、**改动怎么上线**。
你完全不需要懂编程,照着做就行。

---

## 〇、先搞清楚三个词

- **网站**:别人打开网址就能看到的公开页面(首页 / 报告 / 价格 / 交易日志 / 关于我)。
- **后台**:只有你能登录的管理界面,网址是 `你的网址/admin`。你在这里打字、点“发布”。
- **GitHub**:你所有内容永久保存的“云端仓库”。你发布的每一篇、每一次改价格,它都永久存着,还留着完整历史,丢不了。

**你日常只跟“后台”打交道。** 打字 → 点发布 → 几分钟后网站自动更新。就这么简单。

---

## 一、怎么发一篇报告(每日简报 / 周报)

1. 浏览器打开 **`你的网址/admin`**,用 GitHub 账号登录。
2. 左边点 **“报告”** → 右上角点 **“New 报告 / 新建”**。
3. 填写:
   - **标题**:比如 `每日简报 · 2026-07-10`
   - **日期**:点一下选当天(列表就是按这个日期从新到旧排的)
   - **摘要**(可选):一句话,会显示在报告列表里
   - **正文**:直接打字。想加粗、列表、表格,用上面的工具条即可。
4. 点右上角 **“Publish / 发布”**。
5. 等几分钟,打开网站的“报告”页,新报告就在最上面了。

> **交易日志**的做法**完全一样**:左边改点“交易日志”即可。

---

## 二、怎么更新价格(从本地平台导出)

价格页的数据来自你**本地的研究平台**(Daily Update 工作区)。更新分两步:先在平台刷新数据,再导出到网站。

1. 打开你的本地价格平台,像平常一样**刷新数据**(拉到最新)。
2. 回到项目文件夹,**双击 `导出价格到网站.bat`**。
   - 它会把平台里 Daily Update 的所有面板,导出成网站用的 `website\data\prices.json`。
   - 看到 “Done.” 就是成功了。
3. 把 **`website\data\prices.json`** 这个文件上传到 GitHub(见下方第三节)。
4. 几分钟后,网站价格页就更新成最新数据了(分区、折线、涨跌都自动算好)。

> 说明:价格文件是**自动生成**的,请**不要手动改** `prices.json`,也不要在后台改价格——否则下次导出会覆盖掉。
> 想改“搬哪些图/分区”,告诉维护者调整 `tools\export_prices.py` 即可。

---

## 三、改动怎么上线(其实全自动)

你在后台点“发布”后,背后会自动完成:内容存进 GitHub → 网站自动重新生成 → 几分钟后线上更新。
**你什么都不用管,刷新网页就能看到新内容。** 偶尔慢一两分钟是正常的,等一下再刷新即可。

---

## 四、（备用)不想用后台,直接改文件也行

如果哪天你想直接改文件(比如在 GitHub 网页上改),对应关系是:

- 报告:`reports/` 文件夹里,每篇一个 `.md` 文件。新增就照着已有的复制一份改内容;文件名建议 `日期-report.md`。
- 交易日志:`trades/` 文件夹,同理。
- 价格:`data/prices.json`,给对应品种的 `历史` 里加一行 `{ "日期": "2026-07-10", "数值": 526.0 }`。
- 关于我 / 首页文案:`about.html` / `index.html` 里的文字。

> 报告/交易日志的“目录清单”(`index.json`)**不用你管**,每次发布会自动生成。

**每篇报告开头那几行 `--- ... ---` 是什么?** 那是标题、日期、摘要的登记信息,照着已有的格式填即可。例如:

```
---
title: 每日简报 · 2026-07-10
date: 2026-07-10
summary: 今天的一句话摘要
---

正文从这里开始……
```

## 五、常见问题

- **我发布后网站没变?** 等 2~3 分钟再刷新;还不行就再确认后台是否点了“发布”。
- **内容会不会丢?** 不会。GitHub 永久保存 + 完整历史,连你上周改过什么都能翻出来。
- **别人能改我的东西吗?** 不能。后台要用你的 GitHub 账号登录才能进,访客只能看。
- **想改颜色/标题?** 告诉维护者(或按“备用”一节改文字);涨跌颜色在 `assets/style.css` 里 `--up`(涨)`--down`(跌)两行。

---

## 六、首次上线要做的一次性设置(只做一次)

这部分只在**第一次**做,之后就再也不用碰了。详细分步指引见另一份《上线操作指引》,大致是:

1. 注册 GitHub,新建一个仓库,把本文件夹内容传上去。
2. 注册 Netlify,连接这个仓库,自动发布,拿到网址。
3. 配置一次“后台登录”(把 GitHub 和后台打通)。
4. 把 `admin/config.yml` 里的 `repo:` 改成你的 `用户名/仓库名`。

完成后,你就只剩“打字 → 发布”这一件事了。
